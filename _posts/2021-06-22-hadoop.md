# 3. 하둡 설정하기

### 3-1. 환경변수 설정하기

환경변수를 설정해준다.

```python
vim /etc/profile
```

파일안 맨밑에 

```python
export JAVA_HOME=/usr/lib/jvm/java-1.8.0-openjdk-1.8.0.292.b10-1.el7_9.x86_64
export HADOOP_HOME=/root/hadoop
export PATH=$PATH:$JAVA_HOME/bin:$HADOOP_HOME/bin:$HADOOP_HOME/sbin

export HDFS_NAMENODE_USER="root"
export HDFS_DATANODE_USER="root"
export HDFS_SECONDARYNAMENODE_USER="root"
export YARN_RESOURCEMANAGER_USER="root"
export YARN_NODEMANAGER_USER="root"
```

위 코드로 수정해준다.

저장하고 나와서 

```python
source /etc/profile
```

최종적으로는 /etc/profile에 다음줄들이 추가되있다.

![image](https://user-images.githubusercontent.com/48349693/122910639-64cff100-d391-11eb-9279-6cb4b8d4756e.png)

HADOOP_HOME은 편의상이고

JAVA_HOME은 hadoop이 JAVA_HOME 환경변수를 참조하기때문에 필수이다.

---

### 3-2. ssh 키 생성하기

root 홈 디렉터리로 온다.

```python
ssh-keygen
```

위 명령어를 실행한다. ssh-keygen은 엔터만 친다.

그리고 아래 명령어를 실행한다.

(아래 명령어는 나중에 서버(slave)를 추가할 때 사용한다. 지금은 master서버를 다루고 있다.)

```python
ssh-copy-id -i ~/.ssh/id_rsa.pub root@localhost
```

그러면 ssh root@(주소) 를 쳤을때 비밀번호 없이 해당 클라이언트에 접속할 수 있다.

---

### 3-3. 설정파일 수정하기

현재 디렉토리를 변경한다.

```python
cd ~/hadoop/etc/hadoop
```

<span style="color:blue">hadoop-env.sh</span>

해당 파일을 연다.

```python
vim hadoop-env.sh
```

54번줄 주석을 풀고 JAVA_HOME을 수정한다.

~~왜 또 JAVA_HOME 넣는게 필요하지~~

```python
export JAVA_HOME=/usr/lib/jvm/java-1.8.0-openjdk-1.8.0.292.b10-1.el7_9.x86_64
```

<span style="color:blue">core-site.xml</span>

해당파일을 다음과 같이 수정한다.

```python
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
  <property>
    <name>fs.defaultFS</name>
    <value>hdfs://192.168.127.132:9000</value>
  </property>
</configuration>
```

ip 주소 부분은 꼭 master 서버의 주소를 넣어야한다. 

나중에 master서버를 복사할 것이기 때문에 localhost로 쓰면 안된다.

<span style="color:blue">hdfs-site.xml</span>

해당파일을 다음과 같이 수정한다.

```python
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
  <property>
    <name>dfs.replication</name>
    <value>2</value>
  </property>
  <property>
    <name>dfs.permissions</name>
    <value>false</value>
  </property>
  <property>
    <name>dfs.datanode.data.dir</name>
    <value>$HADOOP_HOME/datanode</value>
  </property>
</configuration>
```

숫자 2은 파일 복제 수 이다.

여러개의 서버가 있을때 보통은 3을 사용한다.

<span style="color:blue">mapred-site.xml</span>

해당 파일을 다음과 같이 수정한다.

```python
<?xml version="1.0"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
  <property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
  </property>
</configuration>
```

<span style="color:blue">yarn-site.xml</span>

해당 파일을 다음과 같이 수정한다.

```python
<?xml version="1.0"?>

<configuration>
  <property>
    <name>yarn.nodemanager.aux-services</name>
    <value>mapreduce_shuffle</value>
  </property>
  <property>
    <name>yarn.nodemanager.auxservices.mapreduce.shuffle.class</name>
    <value>org.apache.hadoop.mapred.ShuffleHandler</value>
  </property>
  <property>
    <name>yarn.nodemanager.vmem-check-enabled</name>
    <value>false</value>
  </property>
  <property>
    <name>yarn.application.classpath</name>
    <value>
      $HADOOP_HOME/share/hadoop/mapreduce/*
      $HADOOP_HOME/share/hadoop/mapreduce/lib/*
      $HADOOP_HOME/share/hadoop/common/*
      $HADOOP_HOME/share/hadoop/common/lib/*
      $HADOOP_HOME/share/hadoop/hdfs/*
      $HADOOP_HOME/share/hadoop/hdfs/lib/*
      $HADOOP_HOME/share/hadoop/yarn/*
      $HADOOP_HOME/share/hadoop/yarn/lib/*
    </value>
  </property>
</configuration>
```

# 4. 서버 늘리기

여기서 스냅샷을 찍었다.

사실상 여기까지 한것들은 slave 서버들을 설정해준것이다.

이제껏 설정한것들을 서버복제를 통해서 slave들을 만들고 slave서버를 건들일필요 없다.

일단 master 서버를 종료해준다.

가상머신이 들어있는 폴더를 열고 전체 복사를 해준다.

![image](https://user-images.githubusercontent.com/48349693/122924332-9b613800-d3a0-11eb-847c-ca0d8c49944f.png)

그리고 각 slave 빈폴더에 붙여넣기 해준다.

![image](https://user-images.githubusercontent.com/48349693/122925023-66091a00-d3a1-11eb-9c4e-3a48d6efb59d.png)

그리고 각 폴더안 .vmx 파일을 실행시켜준다.

이왕이면 순서대로 서버들을 가동한다.

![image](https://user-images.githubusercontent.com/48349693/122925517-dd3eae00-d3a1-11eb-841d-9ddf1841b313.png)

아 참고로 컴퓨터의 저장공간을 아끼기위해서 master서버 하나만 복사후 스냅샷을 비운후 스냅샷을 비운 slave서버를 다시 복사해준다.

이왕이면 차후 서버 늘리기 편하게 가동하지않는 복사용 slave 서버를 하나 쟁겨둔다.

# 5. master 서버 최종 설정

일단 root계정 로그인 후 홈 디렉터리로 가준다.

다음 명령을 통해 각 slave서버에 키를 공유해준다.

```python
ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.127.133
ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.127.134
ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.127.135
```

참고로 세 주소는 slave 서버들의 주소이다.

다음 명령을 통해 현재 디렉토리를 바꿔준다.

```python
cd ~/hadoop/etc/hadoop
```

그리고 다음 파일을 수정한다.

<span style="color:blue">hadoop-env.sh</span>

252번줄에 다음 코드로 수정한다.

```python
export HADOOP_SECURE_PID_DIR=$HADOOP_HOME/pids
```

그리고 다음 파일을 수정한다.

<span style="color:blue">hdfs-site.xml</span>

```python
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
  <property>
    <name>dfs.replication</name>
    <value>2</value>
  </property>
  <property>
    <name>dfs.permissions</name>
    <value>false</value>
  </property>
  <property>
    <name>dfs.namenode.name.dir</name>
    <value>$HADOOP_HOME/namenode</value>
  </property>
</configuration>
```

'data'라는 글자를 'name'으로만 바꿔주면 된다.

그리고 다음 파일을 수정한다.

<span style="color:blue">workers</span>

```python
192.168.127.133
192.168.127.134
192.168.127.135
```

localhost 라는 글자가 있었을 텐데 다지우고 slave 서버들의 주소를 넣으면 된다.

이제 앞으로 서버를 늘릴때 (예비용 slave를 복사할때) 

키공유와 workers 파일만 수정해주면 된다.

여기서 master서버를 스냅샷 찍었다.

master 서버에서 다음 명령을 실행해준다.

```python

start-all.sh
```
