# 3. 하둡 설정하기

### 3-1. 환경변수 설정하기

.bashrc를 수정해준다.

```python
vim ~/.bashrc
```

파일안 맨밑에 

```python
export JAVA_HOME=/usr/lib/jvm/java-1.8.0-openjdk-1.8.0.292.b10-1.el7_9.x86_64
export HADOOP_HOME=/home/user/hadoop
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export HADOOP_CONF_DIR=$HADOOP_HOME/etc/hadoop

export HADOOP_COMMON_HOME=$HADOOP_HOME
export HADOOP_CONF_DIR=$HADOOP_HOME/etc/hadoop
export HADOOP_HDFS_HOME=$HADOOP_HOME
export HADOOP_MAPRED_HOME=$HADOOP_HOME
export HADOOP_YARN_HOME=$HADOOP_HOME
```

위 코드를 추가해준다.

저장하고 나와서 

```python
source .bashrc
```

---

### 3-2. 설정파일 수정하기

현재 디렉토리를 변경한다.

```python
cd ~/hadoop/etc/hadoop
```

<span style="color:blue">hadoop-env.sh</span>

해당 파일을 연다.

```python
vim hadoop-env.sh
```

54번줄 주석을 풀고 JAVA_HOME을 수정한다.

```python
export JAVA_HOME=/usr/lib/jvm/java-1.8.0-openjdk-1.8.0.292.b10-1.el7_9.x86_64
```

<span style="color:blue">core-site.xml</span>

해당파일을 다음과 같이 수정한다.

```python
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
  <property>
    <name>fs.defaultFS</name>
    <value>hdfs://master:9000</value>
  </property>
</configuration>
```

<span style="color:blue">hdfs-site.xml</span>

해당파일을 다음과 같이 수정한다.

```python
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
  <property>
    <name>dfs.namenode.name.dir</name>
    <value>/home/user/dfs/nameNode</value>
  </property>
  <property>
    <name>dfs.datanode.data.dir</name>
    <value>/home/user/dfs/dataNode</value>
  </property>
  <property>
    <name>dfs.replication</name>
    <value>2</value>
  </property>
</configuration>
```

숫자 2은 파일 복제 수 이다.

서버가 많이 있을때 보통은 3을 사용한다.

<span style="color:blue">yarn-site.xml</span>

해당 파일을 다음과 같이 수정한다.

```python
<?xml version="1.0"?>

<configuration>
  <property>
    <name>yarn.resourcemanager.hostname</name>
    <value>master</value>
  </property>
</configuration>
```

# 5. master 서버 최종 설정

홈 디렉터리로 가준다.

ssh-keygen 명령을 통해 키를 생성해준다.


다음 명령을 통해 모든 서버에 키를 공유해준다.

```python
ssh-copy-id -i ~/.ssh/id_rsa.pub user@master
ssh-copy-id -i ~/.ssh/id_rsa.pub user@slave1
ssh-copy-id -i ~/.ssh/id_rsa.pub user@slave2
```
그러면 ssh user@(주소) 를 쳤을때 비밀번호 없이 해당 클라이언트에 접속할 수 있다.


다음 명령을 통해 현재 디렉토리를 바꿔준다.

```python
cd ~/hadoop/etc/hadoop
```

그리고 다음 파일을 수정한다.

<span style="color:blue">workers</span>

```python
slave1
slave2
```

localhost 라는 글자가 있었을 텐데 다지우고 slave 서버들의 주소를 넣으면 된다.

이제 앞으로 서버를 늘릴때 (예비용 slave를 복사할때) 

키공유와 workers 파일만 수정해주면 된다.
